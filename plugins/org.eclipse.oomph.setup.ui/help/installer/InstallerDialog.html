This dialog lets you install complete development environments for selected projects, or more exactly project branches, on your local disk. You are only four steps away from being able to work on your favorite projects' code:

Check the Project Branches

In the upper part of the dialog expand the projects you want to work on and check one or more of their branches. Existing installations are shown in blue and can be reinstalled.

Select the Eclipse Versions

For each installation you can select the Eclipse version you want to use as your IDE. The availability of the versions can be restricted by the selected project branch. By default the highest available version is selected. Please note that the selected Eclipse version does not impact PDE's target platform if the setup model includes tasks to provision a separate, dedicated target platform.

Enter the Install Folder

The installer enforces a well-defined and recognizable file system structure under the common install folder. This structure consists of project folders which contain branch folders. Each branch folder contains a separate Eclipse installation which is connected to a single, shared bundle pool. The resulting structure can vary slightly depending on the configurations of the installed projects. Typically project and branch configurations add folders for workspaces, target platforms, API baselines and so on.

By default the install folder is your home folder. You can change it to any location on your disk. It is recommended to use short paths without spaces or other characters that may be critical on your platform.

Start the Installation

When you have checked the project branches to install, selected the respective eclipse versions and entered the appropriate installation folders you may want to edit your user tasks or the network connection settings.

Then click the Install button to start the installation process.

The process starts to install the selected Eclipse versions and the required tools of the checked project branches. The progress of the installation process is reported in a popup dialog that shows the setup tasks to be executed in the upper part and the installation log in the lower part. The setup task currently being executed is always highlighted.

Once an Eclipse IDE is properly installed it is automatocally launched to complete the setup tasks that require to execute within that IDE. These tasks are always automatically executed when that IDE starts up but only if they are actually needed. The progress of the execution is reported in a popup dialog very similar to the one described above.

Within the installed IDE you can use the following buttons on the main toolbar:

Re-execute all needed setup tasks within the current IDE.
Edit your user tasks.
View the configuration of the current project branch.

Edit the User Tasks

You can define any number of setup tasks in your user scope. These tasks can optionally be restricted to specific projects or branches. All user-scoped tasks that are unrestricted or restricted to the current branch are merged into the list of tasks to be executed when the IDE is installed or (re-) configured.

Refer to the Author's Guide for details about setup authoring tools, the different setup tasks, their properties and how they execute.

Update the Installer

When the installer starts it checks for updates in the background. This is indicated by an animated hour glass that overlays the disabled Update button at the bottom of the installer dialog.

When the update check is finished and if no updates were found the hour glass disappears and the button stays disabled. If updates are available the button is enabled.

If no updates were found the Update button is enabled. Clicking on it starts the update process that, once finished, asks you to restart the installer.

Please note that the installer may require you to install updates when it loads its configuration files from remote and detects that they were created by a newer version of the setup tool.
